================================================================================
  1. OVERVIEW
================================================================================

    libhobbycad is the core library for HobbyCAD, providing:

    - Document and project management
    - 3D geometry storage (TopoDS_Shape from OpenCASCADE)
    - Sketch serialization (entities, constraints)
    - Parametric feature tree
    - BREP file I/O
    - .hcad project format (directory-based, git-friendly)
    - 2D geometry operations (intersections, transforms)
    - Sketch operations (offset, fillet, chamfer, trim, patterns)
    - Profile detection for closed loops
    - Snap point detection with weighted priority system

    The library is designed to be used by both GUI and CLI applications.

    ALTERNATIVE UI SUPPORT
    ----------------------
    All functions in libhobbycad are UI-agnostic and can be used to build
    alternative user interfaces. The library has no dependencies on Qt or
    any GUI framework. Core types (Point2D, Rect2D, Vec3) are defined in
    hobbycad/types.h. When Qt 6.4.2+ is available, implicit conversion
    operators provide seamless interop with QPointF, QRectF, and QVector3D.

    The geometry and sketch modules provide all the computational logic needed
    for a 2D sketching application:

    - Entity creation and manipulation (sketch/entity.h)
    - Constraint management (sketch/constraint.h)
    - Geometric operations like fillet, chamfer, offset (sketch/operations.h)
    - Pattern creation (sketch/patterns.h)
    - Profile/loop detection for extrusion (sketch/profiles.h)
    - Snap point detection with weighted priority (sketch/snap.h)
    - Low-level geometry calculations (geometry/*.h)

    To build an alternative UI (e.g., web-based, terminal, or different
    toolkit), simply link against libhobbycad and use the provided functions.
    See Section 13 (GUI Integration) for examples of how the Qt-based
    SketchCanvas converts between its UI types and library types.

    Header Files:
      hobbycad/types.h                Core types (Point2D, Rect2D, Vec3, helpers)

  2.1  Initialization Functions
  ------------------------------

    #include <hobbycad/core.h>

    const char* hobbycad::version()
        Returns the library version string (e.g., "0.0.1").

    bool hobbycad::initialize()
        Initialize the OCCT kernel and library-wide state.
        Call once at application startup before using other functions.
        Returns true on success.

    void hobbycad::shutdown()
        Shut down the library and release resources.
        Call once at application exit.

    Example Usage:

        #include <hobbycad/core.h>

        int main() {
            if (!hobbycad::initialize()) {
                return 1;  // Initialization failed
            }

            // ... use the library ...

            hobbycad::shutdown();
            return 0;
        }


  2.2  Export Macros
  -------------------

    HOBBYCAD_EXPORT
        Macro for exporting symbols from the shared library.
        Automatically handles Windows (__declspec) and Unix (visibility).

    Build Definitions:
        HOBBYCAD_SHARED     Defined when building/linking shared library
        HOBBYCAD_BUILDING   Defined when building the library itself

  ----------------------

        Returns all shapes in the document.

        Add a shape to the document. Marks document as modified.

    void Document::clear()
        Remove all shapes. Marks document as modified.

        Create a default test solid (a box) for initial display.
        Clears existing shapes and adds a single box.

  --------------

  5. DATA STRUCTURES
================================================================================

  5.1  Sketch Entity Types
  -------------------------

    enum class SketchEntityType {
        Point,       // Single point
        Line,        // Line segment (2 points)
        Rectangle,   // Axis-aligned rectangle (2 corner points)
        Circle,      // Circle (center point + radius)
        Arc,         // Arc (center + radius + start/sweep angles)
        Spline,      // Catmull-Rom spline (multiple control points)
        Polygon,     // Regular polygon (center + radius + sides)
        Slot,        // Linear slot (2 arc centers + radius) or
                     // Arc slot (arc center + start + end + radius + arcFlipped)
        Ellipse,     // Ellipse (center + major/minor radii)
        Text,        // Text annotation
        Dimension    // Dimension annotation
    };


  5.2  Constraint Types
  ----------------------

    enum class ConstraintType {
        // Dimensional constraints
        Distance,      // Linear distance between two points or point-to-line
        Radius,        // Circle/arc radius
        Diameter,      // Circle/arc diameter
        Angle,         // Angle between two lines

        // Geometric constraints
        Horizontal,    // Line is horizontal
        Vertical,      // Line is vertical
        Parallel,      // Two lines are parallel
        Perpendicular, // Two lines are perpendicular
        Coincident,    // Two points share same position
        Tangent,       // Arc/circle tangent to line or arc/circle
        Equal,         // Two entities have equal length/radius
        Midpoint,      // Point at midpoint of line
        Symmetric      // Two points symmetric about a line
    };


  5.3  Plane Types
  -----------------

    enum class SketchPlane {
        XY,      // XY plane (normal along Z)
        XZ,      // XZ plane (normal along Y)
        YZ,      // YZ plane (normal along X)
        Custom   // Custom angled plane or references a ConstructionPlane
    };

    enum class PlaneRotationAxis {
        X,
        Y,
        Z
    };

    enum class ConstructionPlaneType {
        OffsetFromOrigin,    // Offset from XY, XZ, or YZ origin plane
        OffsetFromPlane,     // Offset from another construction plane
        Angled               // Rotated around one or two axes
    };


  5.4  Feature Types
  -------------------

    enum class FeatureType {
        Origin,      // Coordinate system origin
        Sketch,      // 2D sketch
        Extrude,     // Extrusion of sketch profile
        Revolve,     // Revolution of sketch profile
        Fillet,      // 3D fillet (rounded edges)
        Chamfer,     // 3D chamfer (beveled edges)
        Hole,        // Hole feature
        Mirror,      // Mirror operation
        Pattern,     // Linear or circular pattern
        Box,         // Primitive box
        Cylinder,    // Primitive cylinder
        Sphere,      // Primitive sphere
        Move,        // Move/transform operation
        Join,        // Boolean union
        Cut,         // Boolean subtraction
        Intersect    // Boolean intersection
    };


  5.5  Sketch Data Structures
  ----------------------------

    struct SketchEntityData {
        int id = 0;                              // Unique entity ID
        SketchEntityType type;                   // Entity type
        std::vector<Point2D> points;                 // Control/definition points
        double radius = 0.0;                     // For circles, arcs, slots
        double startAngle = 0.0;                 // For arcs (degrees)
        double sweepAngle = 0.0;                 // For arcs (degrees)
        int sides = 6;                           // For polygons
        double majorRadius = 0.0;                // For ellipses
        double minorRadius = 0.0;                // For ellipses
        std::string text;                            // For text entities
        bool constrained = false;                // Has active constraints
        bool isConstruction = false;             // Construction geometry flag
    };

    struct ConstraintData {
        int id = 0;                              // Unique constraint ID
        ConstraintType type;                     // Constraint type
        std::vector<int> entityIds;                  // IDs of entities involved
        std::vector<int> pointIndices;               // Point indices within entities
        double value = 0.0;                      // Dimension value
        bool isDriving = true;                   // Driving vs reference
        Point2D labelPosition;                   // Label position in sketch
        bool labelVisible = true;                // Label visibility
        bool enabled = true;                     // Constraint active state
    };

    struct SketchData {
        std::string name;                            // Sketch name
        SketchPlane plane = SketchPlane::XY;     // Plane type
        int constructionPlaneId = -1;            // Construction plane reference
        double planeOffset = 0.0;                // Offset from origin
        PlaneRotationAxis rotationAxis;          // Rotation axis
        double rotationAngle = 0.0;              // Rotation angle
        std::vector<SketchEntityData> entities;      // All entities
        std::vector<ConstraintData> constraints;     // All constraints
        double gridSpacing = 10.0;               // Grid spacing (mm)
    };


  5.6  Construction Plane Data
  -----------------------------

    struct ConstructionPlaneData {
        int id = 0;                              // Unique plane ID
        std::string name;                            // User-visible name
        ConstructionPlaneType type;              // How plane is defined

        // Base plane reference
        SketchPlane basePlane = SketchPlane::XY; // For OffsetFromOrigin
        int basePlaneId = -1;                    // For OffsetFromPlane

        // Plane origin in absolute coordinates
        double originX = 0.0;
        double originY = 0.0;
        double originZ = 0.0;

        // Offset along normal
        double offset = 0.0;

        // Primary rotation
        PlaneRotationAxis primaryAxis;
        double primaryAngle = 0.0;               // Degrees

        // Secondary rotation
        PlaneRotationAxis secondaryAxis;
        double secondaryAngle = 0.0;             // Degrees

        // Roll (rotation around normal)
        double rollAngle = 0.0;                  // Degrees

        // Visibility
        bool visible = true;

        bool hasCustomOrigin() const;            // Non-zero origin check
    };


  5.7  Parameter Data
  --------------------

    struct ParameterData {
        std::string name;             // Parameter name (e.g., "length")
        std::string expression;       // Expression string (e.g., "width * 2")
        double value = 0.0;       // Evaluated numeric value
        std::string unit;             // Unit string (e.g., "mm")
        std::string comment;          // User comment
        bool isUserParam = true;  // User vs system parameter
    };


  5.8  Feature State
  -------------------

    enum class FeatureState {
        Normal,      // Feature is valid
        Warning,     // Feature has a warning (e.g., over-constrained sketch)
        Error        // Feature has an error (e.g., failed BREP operation)
    };

    Feature state is determined by library-level operations:
    - Constraint solver results (Inconsistent, RedundantOkay)
    - BREP operation failures (no closed profile, boolean failure)
    - Dependency validation (missing referenced features)


  5.9  Feature Data
  ------------------

    struct FeatureData {
        int id = 0;               // Unique feature ID
        FeatureType type;         // Feature type
        std::string name;             // Feature name
        QJsonObject properties;   // Feature-specific properties
        std::vector<int> dependsOn;   // IDs of features this depends on
        bool suppressed = false;  // Feature is suppressed (not computed)
        FeatureState state;       // Validation state (Normal/Warning/Error)
        std::string stateMessage;     // Human-readable error/warning message
    };

================================================================================
  9. .HCAD FILE FORMAT
================================================================================

    The .hcad format is a directory-based project structure designed for
    git-friendliness and human-readability.

    Directory Structure:

        my_project/
        +-- my_project.hcad          # Project manifest (JSON)
        +-- geometry/
        |   +-- body_001.brep        # 3D geometry (BREP format)
        |   +-- body_002.brep
        +-- planes/
        |   +-- plane_001.json       # Construction plane definitions
        +-- sketches/
        |   +-- sketch_001.json      # Sketch data (entities, constraints)
        |   +-- sketch_002.json
        +-- parameters.json          # User-defined parameters
        +-- features.json            # Feature tree

    Manifest File (project_name.hcad):

        {
            "format_version": 1,
            "hobbycad_version": "0.0.1",
            "name": "My Project",
            "author": "User Name",
            "description": "Project description",
            "units": "mm",
            "created": "2026-02-14T12:00:00Z",
            "modified": "2026-02-14T14:30:00Z",
            "geometry_files": ["geometry/body_001.brep"],
            "construction_plane_files": ["planes/plane_001.json"],
            "sketch_files": ["sketches/sketch_001.json"]
        }

    Sketch File Format:

        {
            "name": "Sketch1",
            "plane": "XY",
            "plane_offset": 0.0,
            "rotation_axis": "X",
            "rotation_angle": 0.0,
            "construction_plane_id": -1,
            "grid_spacing": 10.0,
            "entities": [
                {
                    "id": 1,
                    "type": "Line",
                    "points": [[0, 0], [100, 0]],
                    "is_construction": false
                }
            ],
            "constraints": [
                {
                    "id": 1,
                    "type": "Distance",
                    "entity_ids": [1],
                    "value": 100.0,
                    "is_driving": true,
                    "label_position": [50, 10]
                }
            ]
        }

    Construction Plane File Format:

        {
            "id": 1,
            "name": "Reference Plane 1",
            "type": "OffsetFromOrigin",
            "base_plane": "XY",
            "origin": [0, 0, 50],
            "offset": 0.0,
            "primary_axis": "X",
            "primary_angle": 0.0,
            "secondary_axis": "Y",
            "secondary_angle": 0.0,
            "roll_angle": 0.0,
            "visible": true
        }

    Parameters File Format:

  10.1  Storage Convention
  ------------------------

    **All internal measurements use millimeters (mm) as the base unit.**

    This includes:
      - All sketch entity coordinates and dimensions
      - All constraint values
      - All project file storage (.hcad format)
      - All library function parameters and return values

    Unit conversion happens ONLY at the GUI display layer.

  10.2  Length Units
  ------------------

    enum class LengthUnit {
        Millimeters = 0,  // mm (base unit)
        Centimeters = 1,  // cm
        Meters = 2,       // m
        Inches = 3,       // in
        Feet = 4          // ft
    };

  10.3  Conversion Functions
  --------------------------

    double unitScale(LengthUnit unit)
        Returns the conversion factor (mm per unit).
        Multiply a unit value by this to get mm.

    const char* unitSuffix(LengthUnit unit)
        Returns the display suffix ("mm", "cm", "m", "in", "ft").

    std::string unitSuffixStr(LengthUnit unit)
        Same as unitSuffix, but returns std::string.

    double mmToUnit(double mm, LengthUnit toUnit)
        Converts a value from mm to the specified unit.

    double unitToMm(double value, LengthUnit fromUnit)
        Converts a value from the specified unit to mm.

    double convertLength(double value, LengthUnit from, LengthUnit to)
        Converts between any two units.

    LengthUnit parseUnitSuffix(const std::string& suffix)
        Parses a suffix string to get the unit enum.
        Returns Millimeters if not recognized.

    LengthUnit lengthUnitFromIndex(int index)
        Converts an index (0-4) to a LengthUnit.

    int lengthUnitToIndex(LengthUnit unit)
        Converts a LengthUnit to an index (0-4).

    int unitDisplayPrecision(LengthUnit unit)
        Returns display precision (4 for all units).
        Use for manual formatting without trailing zero trimming.

    std::string formatValue(double value)
        Formats a value for display with up to 4 decimal places.
        Trailing zeros are trimmed (e.g., "1.5000" → "1.5").

    std::string formatStorageValue(double value)
        Formats a value for file storage with 15 decimal places.

    std::string formatValueWithUnit(double mm, LengthUnit unit)
        Converts mm to unit and formats with suffix (e.g., "25.4 mm").

    double parseValueWithUnit(std::string input, LengthUnit defaultUnit)
        Parses "25.4 mm" or "1 in" strings back to mm.
        Returns value in mm; uses defaultUnit if no suffix present.

    bool isMetric(LengthUnit unit)
        Returns true for mm, cm, m.

    bool isImperial(LengthUnit unit)
        Returns true for in, ft.

    constexpr int DisplayPrecision = 4
        Maximum decimal places for display formatting.

    constexpr int StoragePrecision = 15

  12.6  Constraint Solver (solver.h)
  ---------------------------------

    The solver wraps libslvs for parametric constraint solving.

    struct SolveResult

        }

        bool wouldOverConstrain   Would adding constraint fail?
        std::vector<int> conflictingConstraintIds
        std::string reason            Human-readable explanation

    class Solver

        SolveResult solve(entities, constraints)
            Solve constraints and update entity geometry in place.
            Returns result with success status and diagnostics.

        bool wouldOverConstrain(entities, existingConstraints, newConstraint)
            Test if adding a new constraint would over-constrain.

        OverConstraintInfo checkOverConstrain(entities, existing, new)
            Detailed over-constraint check with conflict info.

        int degreesOfFreedom(entities, constraints)
            Calculate remaining DOF (0 = fully constrained).

        static bool isAvailable()
            True if libslvs is compiled in.

    Utility Functions:
        std::string solveResultName(ResultCode code)
        bool constraintSupported(ConstraintType type)
        std::vector<ConstraintType> supportedConstraintTypes()

    Supported Constraint Types:
        Distance, Radius, Diameter, Angle, Horizontal, Vertical,
        Parallel, Perpendicular, Coincident, Equal, Tangent,
        Midpoint, Symmetric

    Example Usage:
        Solver solver;
        std::vector<Entity> entities = { ... };
        std::vector<Constraint> constraints = { ... };

        SolveResult result = solver.solve(entities, constraints);
        if (result.success) {
            // entities have been modified to satisfy constraints
            std::cout << "DOF remaining: " << result.dof << "\n";
        } else {
            std::cerr << "Solve failed: " << result.errorMessage << "\n";
            for (int id : result.failedConstraintIds) {
                // highlight failed constraints in UI
            }
        }

  12.13  Snap Point Detection (snap.h)
  -------------------------------------

    Snap point detection for cursor snapping during sketch editing.
    Uses a weighted distance competition system where all snap
    candidates compete, with higher-weight types having stronger pull.

    Snap Types:

        enum class SnapType {
            Endpoint,       // Line/arc/spline endpoints, rectangle corners
            Point,          // Standalone sketch point (stronger pull)
            Midpoint,       // Line/arc midpoints, edge midpoints
            Center,         // Circle/arc/ellipse/polygon centers
            Quadrant,       // Circle/ellipse axis points (0/90/180/270)
            Intersection,   // Entity-entity or entity-axis crossings
            ArcEndCenter,   // Slot endpoint arc centers
            Nearest,        // Nearest point on entity perimeter
            Origin,         // Origin point (0,0)
            AxisX,          // Point on X axis (Y=0)
            AxisY           // Point on Y axis (X=0)
        };

    Snap Point:

        struct SnapPoint {
            Point2D position;   // World-space position
            SnapType type;      // Type of snap point
            int entityId;       // Source entity ID (-1 for origin/axis)
        };

    Snap Result:

        struct SnapResult {
            bool found;         // Whether a snap point was found
            SnapPoint snap;     // The winning snap point
        };

    Snap Weight:

        double defaultSnapWeight(SnapType type)
            Returns default priority weight for a snap type.
            Higher weight = stronger pull.

            Snap Type       Weight    Description
            ─────────────   ──────    ──────────────────────────────
            Origin          4.0       Strongest pull
            Point           3.0       Standalone sketch points
            Endpoint        2.5       Line/arc/spline endpoints
            Center          2.5       Circle/arc/polygon centers
            Intersection    2.2       Entity and axis crossings
            Midpoint        2.0       Line/arc midpoints
            Quadrant        2.0       Circle/ellipse axis points
            ArcEndCenter    2.0       Slot end arc centers
            AxisX / AxisY   1.25      Nearest point on X or Y axis
            Nearest         1.0       Weakest — perimeter snap

            Formula: effectiveDist = rawDist / weight

    Snap Point Collection:

        std::vector<SnapPoint> collectSnapPoints(const Entity& entity)
            Collect all explicit snap points from a single entity.
            Returns endpoints, midpoints, centers, quadrants, etc.
            depending on entity type.  Does not include intersections
            or nearest-on-perimeter.

        std::vector<SnapPoint> collectAllSnapPoints(
            const std::vector<Entity>& entities,
            int excludeEntityId = -1)
            Collect all snap points from all entities, plus entity-
            entity intersection points and axis-crossing points.
            Pass excludeEntityId to skip a specific entity (e.g.,
            the entity being dragged).

        std::vector<SnapPoint> collectIntersectionSnapPoints(
            const std::vector<Entity>& entities,
            int excludeEntityId = -1)
            Collect intersection points between all entity pairs
            plus axis-crossing points.

        std::vector<SnapPoint> collectAxisCrossingSnapPoints(
            const std::vector<Entity>& entities,
            int excludeEntityId = -1)
            Collect points where entities cross X axis (Y=0) or
            Y axis (X=0).  Handles Line, Rectangle, Parallelogram,
            Circle, Arc, Polygon, and Ellipse.  Skips origin (0,0).

    Nearest on Perimeter:

        SnapPoint findNearestOnPerimeter(
            const std::vector<Entity>& entities,
            const Point2D& point,
            double tolerance,
            int excludeEntityId = -1)
            Find the nearest point on any entity perimeter.
            Handles Line, Circle, Arc, Rectangle, Parallelogram,
            Polygon, and Slot.  Returns SnapPoint with type=Nearest
            if found, or entityId=-1 if nothing within tolerance.

    Best Snap Evaluation:

        SnapResult findBestSnap(
            const std::vector<Entity>& entities,
            const Point2D& worldPos,
            double worldTolerance,
            int excludeEntityId = -1)
            Find the best snap point using weighted distance.
            Checks origin, all explicit snap points, intersections,
            axis crossings, nearest-on-perimeter, and axis proximity.
            Does NOT handle grid snapping (GUI responsibility).

    Entity Intersection Utility:

        std::vector<Point2D> computeEntityIntersectionPoints(
            const Entity& e1, const Entity& e2)
            Compute intersection points between two entities.
            Handles Line, Circle, Arc, Rectangle, Parallelogram,
            and Polygon in all pairwise combinations.  Rectangles,
            parallelograms, and polygons are decomposed into edges.

    Example Usage:

        #include <hobbycad/sketch/snap.h>

        // Simple: find best snap near cursor
        std::vector<Entity> entities = getSketchEntities();
        double tolerance = 10.0 / zoomLevel;  // pixels to world

        SnapResult result = sketch::findBestSnap(
            entities, cursorWorldPos, tolerance);
        if (result.found) {
            cursorPos = result.snap.position;
            showSnapIndicator(result.snap.type);
        }

        // Exclude entity being dragged
        SnapResult result = sketch::findBestSnap(
            entities, cursorWorldPos, tolerance,
            draggedEntityId);

        // Collect snap points for custom evaluation
        auto points = sketch::collectAllSnapPoints(entities);
        for (const auto& sp : points) {
            double dist = QLineF(cursorPos, sp.position).length();
            double effective = dist / sketch::defaultSnapWeight(sp.type);
            // ... custom evaluation logic ...
        }

        #include <hobbycad/qt_compat.h>

        fromQt(qStr)                QString -> std::string

    Legacy Conversion Functions:
        The old toLibraryEntity/toGuiEntity wrappers (in sketchutils.h)

        // Container conversion when Qt API requires QVector

        // String conversion when Qt API requires QString

================================================================================
  14. NON-QT FALLBACK ARCHITECTURE
================================================================================

    libhobbycad is designed to build and function without Qt.  When Qt is
    not available, the library provides equivalent functionality through
    cross-platform alternatives:

      - JSON serialization:  nlohmann/json (header-only, C++11)
      - OpenGL probing:      EGL (optional; stub if unavailable)
      - Base64 encoding:     hobbycad/base64.h (header-only utility)
      - File I/O:            std::filesystem + std::fstream (C++17)

    The existing Qt code paths remain untouched — all fallback code lives
    in #else branches, selected at compile time.

  14.1  HOBBYCAD_HAS_QT Macro
  -----------------------------------------------------------------------

    Defined in types.h (lines 23-35).  Evaluates to 1 when Qt 6.4.2+ is
    detected at compile time, 0 otherwise.  All Qt-conditional code in
    the library uses this single macro:

        #if HOBBYCAD_HAS_QT
        // Qt implementation
        #else
        // Non-Qt fallback
        #endif

    CMake detection (src/libhobbycad/CMakeLists.txt):

        find_package(Qt6 6.4.2 QUIET COMPONENTS Core Gui OpenGL)
        set(HOBBYCAD_LIB_HAS_QT ${Qt6_FOUND})

    When Qt is found, the Qt6::Core target defines QT_CORE_LIB, which
    triggers HOBBYCAD_HAS_QT=1 in types.h.  No explicit CMake define
    is needed.

  14.2  JSON Serialization (nlohmann/json)
  -----------------------------------------------------------------------

    DEPENDENCY:  nlohmann-json (header-only, found via CMake)
    PACKAGE:     nlohmann-json3-dev (Ubuntu), nlohmann-json (vcpkg)
    HEADER:      <nlohmann/json.hpp>

    When Qt is available, the library uses QJsonDocument / QJsonObject /
    QJsonArray for project serialization.  Without Qt, nlohmann/json
    provides identical functionality with a nearly 1:1 API mapping.

    FILES WITH DUAL IMPLEMENTATIONS:

      project.cpp       Full project save/load (manifest, sketches,
                         construction planes, parameters, features,
                         foreign files).  Non-Qt path uses
                         nlohmann::json + std::fstream with
                         std::filesystem::path for cross-platform
                         path handling.

      parameters.cpp    ParameterEngine::toJson() / fromJson().
                         Qt path returns QJsonObject; non-Qt path
                         returns nlohmann::json.

    API TRANSLATION TABLE:

      Qt API                              nlohmann equivalent
      ----------------------------------  ----------------------------------
      QJsonObject obj;                    nlohmann::json obj;
      obj["key"] = value;                 obj["key"] = value;
      json["key"].toString().toStdString()  json.value("key", std::string{})
      json["key"].toInt(default)          json.value("key", default)
      json["key"].toDouble(default)       json.value("key", default)
      json["key"].toBool(default)         json.value("key", default)
      json["key"].toArray()               json["key"] (iterate directly)
      json.contains("key")               json.contains("key")
      QJsonArray arr; arr.append(x)       json arr = json::array();
                                          arr.push_back(x)
      QJsonDocument(obj).toJson(Indent)   obj.dump(4)
      QJsonDocument::fromJson(bytes)      nlohmann::json::parse(stream)
      QFile + readAll()                   std::ifstream + istreambuf_iter
      QFile + write()                     std::ofstream << json.dump(4)

    ERROR HANDLING:  nlohmann::json throws parse_error on invalid JSON.
    All load functions wrap json::parse() in try/catch and convert to
    error strings via the errorMsg parameter.

    FeatureData::properties:
      With Qt:     QJsonObject properties;
      Without Qt:  nlohmann::json properties = nlohmann::json::object();
      Both support bracket-access, iteration, and .empty() checks.

  14.3  OpenGL Probing (EGL)
  -----------------------------------------------------------------------

    DEPENDENCY:  EGL (optional; libEGL-dev on Ubuntu, part of Mesa)
    CMAKE:       find_package(OpenGL COMPONENTS EGL QUIET)
    DEFINE:      HOBBYCAD_HAS_EGL=1 when found
    HEADER:      hobbycad/opengl_info.h

    The OpenGLInfo struct and probeOpenGL() function are always
    available regardless of Qt.  Three implementation paths exist:

      1. Qt path (HOBBYCAD_HAS_QT):
         Uses QOpenGLContext + QOffscreenSurface.
         Requires QApplication to exist before calling.

      2. EGL path (HOBBYCAD_HAS_EGL, no Qt):
         Creates a headless EGL pbuffer context.
         No window system or application object required.
         Works on Linux (Mesa, NVIDIA), and any platform with EGL.

      3. Stub path (no Qt, no EGL):
         Returns OpenGLInfo with contextCreated=false and
         errorMessage="OpenGL probing not available (no Qt or EGL)".

    The summary() method always returns std::string (not QString).
    GUI code wraps with QString::fromStdString() as needed.

    EGL PROBING SEQUENCE:
      1. eglGetDisplay(EGL_DEFAULT_DISPLAY)
      2. eglInitialize()
      3. eglBindAPI(EGL_OPENGL_API)  — desktop GL, not GL ES
      4. eglChooseConfig() with EGL_PBUFFER_BIT + EGL_OPENGL_BIT
      5. eglCreatePbufferSurface() — 1x1 pixel
      6. eglCreateContext() — try 3.3 Core, fallback to unconstrained
      7. glGetString(GL_VERSION / GL_RENDERER / GL_VENDOR / GL_GLSL)
      8. Parse major.minor from version string
      9. Cleanup: destroy context, surface, terminate display

  14.5  Translation / i18n
  -----------------------------------------------------------------------

    The library (libhobbycad) is intentionally translation-free.  All
    user-visible strings use std::string, not tr().  This ensures the
    library compiles and works identically with or without Qt.

    Translation is a GUI-only concern:
      - All tr() calls are in src/hobbycad/ (the application)
      - Translation files (.ts) live in translations/
      - CMake target: hobbycad_lupdate (Qt LinguistTools)
      - Runtime loading via QTranslator in main.cpp

    One placeholder exists in the library:

      sketch/undo.h:  entityTypeDisplayName(EntityType)
        Currently returns the plain English name (entityTypeName).
        Documented as a future hook for tr() integration, but
        not yet implemented.

  14.6  Cross-Platform Considerations
  -----------------------------------------------------------------------

    All non-Qt fallback code is cross-platform:

      nlohmann/json:      C++11 header-only, all platforms.
      base64.h:           Pure C++ standard library, all platforms.
      std::filesystem:    C++17, supported by GCC 8+, Clang 7+, MSVC 19.14+.
      EGL:                Linux (Mesa, NVIDIA), Windows (ANGLE), macOS (limited).

    PATH HANDLING:
      The non-Qt file I/O code uses std::filesystem::path operators
      (operator/) for all path construction, ensuring correct separators
      on all platforms:

        (fs::path(dir) / "features" / "parameters.json").string()

      Relative paths stored in JSON (e.g., "sketches/sketch_001.json")
      use forward slashes per JSON convention; fs::path handles
      conversion when constructing absolute paths.

    EGL AVAILABILITY:
      - Linux:   Standard (libEGL-dev / libegl1-mesa-dev)
      - Windows: Available via ANGLE (used by Chrome, Electron)
      - macOS:   Limited; CGL is the native alternative

      When EGL is not found, the stub fallback ensures the build
      succeeds.  probeOpenGL() returns a meaningful error message
      rather than failing silently.

  14.8  Background Image Support (stb_image + libwebp)
  -----------------------------------------------------------------------

    When Qt is not available, background image loading and manipulation
    are provided by stb_image (a public-domain single-header library)
    with optional WebP support from libwebp.

    COMPILE-TIME MACROS:

        by CMake.  Guards all stb-dependent code paths.
      - HOBBYCAD_HAS_WEBP       — defined to 1 when libwebp is found
        by pkg-config.  Enables WebP decoding as a fallback after
        stb_image in loadImageFromMemory()/queryImageDimensionsFromMemory().

    FILES:
      - hobbycad/image_buffer.h  — ImageBuffer struct (RGBA8 pixel buffer)
                                   and loading/manipulation function
                                   declarations
      - image_buffer.cpp         — stb_image-based implementation
                                   (#define STB_IMAGE_IMPLEMENTATION),
                                   with #if HOBBYCAD_HAS_WEBP fallback
                                   using WebPDecodeRGBA()/WebPGetInfo()
      - sketch/background.cpp    — Non-Qt #elif blocks using ImageBuffer

    ImageBuffer TYPE:
      A simple RGBA8 pixel buffer with direct channel accessors:

        struct ImageBuffer {
            std::vector<uint8_t> pixels;  // width * height * 4 bytes
            int width, height;
            bool isValid() const;
            uint8_t red(x, y), green(x, y), blue(x, y), alpha(x, y);
            void setPixel(x, y, r, g, b, a);
            static uint8_t grayValue(r, g, b);  // ITU-R BT.601
        };

    SUPPORTED FORMATS:
      stb_image: PNG, JPEG, BMP, GIF, TGA, PSD, HDR, PNM
      libwebp:   WebP (optional, requires libwebp-dev)

      Note: Qt's QImage also requires the qtimageformats plugin for
      WebP.  This is listed in vcpkg.json and checked at CMake
      configure time.  On Ubuntu: sudo apt install
      qt6-image-formats-plugins

    API COMPARISON (Qt vs stb_image path):

      | Qt path                        | stb_image path                  |
      |--------------------------------|---------------------------------|
      | QImage getBackgroundQImage()   | ImageBuffer getBackgroundImage()|
      | QImage applyBackgroundAdj()    | ImageBuffer applyBackgroundAdj()|
      | QImageReader for dimensions    | queryImageDimensions()          |
      | QImage::loadFromData()         | loadImageFromMemory()           |
      | QImage::mirrored()             | flipHorizontal/flipVertical()   |

    LOADING PIPELINE (non-Qt):
      loadImageFromMemory(data, length):
        1. Try stb_image (PNG/JPEG/BMP/GIF/TGA/PSD/HDR/PNM)
        2. If stb fails and HOBBYCAD_HAS_WEBP: try WebPDecodeRGBA()
        3. Return empty ImageBuffer if both fail

      queryImageDimensionsFromMemory(data, length):
        1. Try stbi_info_from_memory()
        2. If stbi fails and HOBBYCAD_HAS_WEBP: try WebPGetInfo()

    GRACEFUL DEGRADATION:
      If stb_image is not installed (HOBBYCAD_HAS_STB_IMAGE=0):
      - loadBackgroundImage() still works for file/embed operations
        but cannot detect image dimensions (originalPixelWidth/Height
        remain 0, default size used)
      - getBackgroundImage() and applyBackgroundAdjustments() are not
        available (declarations excluded from background.h)
      - Serialization (backgroundToJson/backgroundFromJson) still works
        via nlohmann/json + base64 encoding

      If libwebp is not installed (HOBBYCAD_HAS_WEBP=0):
      - WebP files will fail to decode (stb_image does not support WebP)
      - All other formats continue to work normally
      - "webp" will not appear in supportedImageFormats()

    INSTALLING OPTIONAL DEPENDENCIES:
      sudo apt install libstb-dev                  # stb_image (non-Qt)
      sudo apt install libwebp-dev                 # WebP support (non-Qt)
      sudo apt install qt6-image-formats-plugins   # WebP support (Qt)

    WebP SUPPORT MATRIX:
      | Build path      | WebP requires                           |
      |-----------------|-----------------------------------------|
      | Qt              | qtimageformats plugin (runtime, no link)|
      | Non-Qt + libwebp| libwebp-dev (linked, HOBBYCAD_HAS_WEBP) |
      | Non-Qt only     | Not available                           |
